# Erstes GitHub-Versionierungsprojekt

## Kurzbeschreibung  
Dieses Projekt dient als Einstieg in die Versionsverwaltung mit Git und GitHub. Ziel ist es, die grundlegenden Funktionen wie Repository-Erstellung, Commits, Branches und Pull Requests kennenzulernen und praktisch anzuwenden.

## Ziel  
- Verständnis für Versionskontrolle entwickeln  
- Arbeiten mit Git-Befehlen erlernen  
- GitHub-Workflow (Commit, Push, Pull Request, Merge) anwenden  
- Erste Erfahrung mit kollaborativer Softwareentwicklung sammeln  

## Workflow 
1. Repository erstellen  
2. Dateien hinzufügen und committen  
3. Änderungen auf GitHub pushen  
4. Branch erstellen und bearbeiten  
5. Pull Request öffnen und mergen
